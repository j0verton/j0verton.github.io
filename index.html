Truncheons & Flagons

This project is a scorekeeper for what was ostensibly a "popular tavern game" in the 1130's. I can't comment on whether that is true, but I can say that for me the project mainly about learning to work with different types of fetch requests to a local API. Nuttshell

I worked in a group of 4 to build a social media application that allowed users to create and share events and articles, to maintain a task list, to view the current weather for their location, and to message other users, both privately and publicly. We built this project twice, once with JS and once with ReactJS.
In the React version I built the component that allowed users to share articles and links, with full CRUD capability. I also worked on the friend relationship, allowing users to add other users as friends from the articles and messaging components.
In the JS version I created the user authentication and messaging components. My messaging component allowed for both private messages and a global chat, as well as including a long polling feature that allowed users to interact in near real time.

B&B Travel Planner

This is a project I completed in September working with two great collaborators. We built a site that allows the user to build a travel itinerary from a list of options complied from 3 APIs (most notably the National Parks API). The goal of our site was to focus on Accessibility options available at the different destinations, so we filtered the data to allow the user to find the information they needed to make decisions about their trip.

This project was really rewarding because of the challenge posed by working with complicated API data (for the first time on this scale) and manipulating it within our Java Script code. I coded several features of the site:

  • saving selected routes to a local API
  • filtering data from the National Parks API to highlight accessibility options on the drop-down menus
  • using another API to create directions for the itinerary
Looking back on this project I can see how I wrestled with asynchronicity. Its funny to see how my code evolved (and hopefully improved)!
